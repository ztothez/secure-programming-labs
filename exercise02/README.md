# Exercise 01 – Insecure C Patterns

This exercise demonstrates intentionally vulnerable C code to illustrate common memory safety issues.

The file `insecure_c_patterns.c` includes examples of:

- Stack buffer overflow
- Off-by-one errors
- Heap overflow
- Function pointer corruption
- Format string vulnerability

Each example is triggered through a simple menu.

The purpose of this exercise is to understand how small mistakes in memory handling can lead to serious security vulnerabilities.

⚠ These vulnerabilities are intentional and for learning purposes only.
